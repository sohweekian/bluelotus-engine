2. ROOT CAUSE FOUND -- TXT vs EXCEL/WORD DIVERGENCE
───────────────────────────────────────────────────────────────
The Word/Excel generator's build_concentration_risk() function
was missing the cluster escalation rule added in the R6 patch.

  TXT (R6):      cluster GOLD_MINERS=66% --> CRITICAL  CORRECT
  Word/Excel:    HHI=0.242 > 0.20 only   --> HIGH      WRONG

Fix: Added identical cluster escalation to build_concentration_risk()
in research_report_generator.py:

  cluster_sev = 'CRITICAL' if cluster_max_val >= 0.65
             or 'HIGH'     if cluster_max_val >= 0.45
  if _SEV_ORDER[cluster_sev] > _SEV_ORDER[status_base]:
      status = cluster_sev

All three formats now report: concentration = CRITICAL
