**BlueLotus V3 — deterministic research engine (frozen v3.0.0)**

Governance gates · contradiction clerk · Bayesian thesis updates (NITE-PEI) · **Nash / QRE game theory (BGTM-V1)** · institutional learning (SLICDO).

## What this software does

| Layer | Function |
|-------|----------|
| **Governance** | Contract validation, approval gate, scenario overlay, regression tests |
| **Reports** | Deterministic TXT / XLSX / DOCX intelligence package |
| **Zone A clerk** | Contradiction map from rules and portfolio math — **no LLM agents** |
| **NITE-PEI** | Bayesian thesis probabilities, CKRI kill-risk, Kelly advisory |
| **BGTM-V1** | Nash Equilibrium · QRE · Correlated Equilibrium · Geo-LR → NITE-PEI |
| **SLICDO** | Claims, resolutions, replay, CIO-gated learning proposals |

**Not included:** GitHub publish, Telegram, broker fetchers, LLM agent council, private portfolio data.

Full explanation: [WHAT_IS_BLUELOTUS.md](https://github.com/sohweekian/bluelotus-engine-docs/blob/main/WHAT_IS_BLUELOTUS.md) · [BGTM game theory](https://github.com/sohweekian/bluelotus-engine-docs/blob/main/BGTM.md)

**In one line:** each pipeline run produces a **full-stack institutional intelligence package** — portfolio truth, market tape, flow, thesis survival, kill risk, governance, contradictions, forecasts, manual CIO authority, and QA — as TXT/DOCX narrative and pivot-ready XLSX.

## Report package capabilities

What the software can produce each cycle, by information type:

### Identity, governance & doctrine
- Cycle and snapshot timestamps (report, dataset, broker, dashboard lag)
- Governance pack binding status, hash, and release verdict
- Execution authority posture (manual-only, no routing, zero system orders)
- Chief Clerk master prompt and source-priority hierarchy
- CIO context capsule (doctrine, sleeve rules, kill conditions, forbidden interpretations)
- SLICDO narrative firewall status (LLM quarantine)
- Data certainty labeling (confirmed vs inferred vs CIO judgment)

### Regime & compressed operating truth
- Market regime label and score
- Regime action guidance
- CIO action compression (wait/hold-style posture)
- Confidence score and blind-spot status
- Causal explanation completeness
- Institutional review / report status flags
- Sentiment hygiene warnings

### Portfolio & reconciliation
- Live truth reconciliation (cash/MV deltas, gold-miner cross-check)
- Total assets, cash, equity, buying power
- Position-level quantity, price, cost, market value, P/L
- Concentration metrics (HHI, top-3 weight, largest name)
- Cluster exposure (e.g. gold miners)
- Mandate tags per position (tactical vs baseline)
- System-generated per-ticker action notes (not CIO orders)
- Open orders list with status, intent, and CIO-review flags
- Recent fills and execution history counts
- Failed or blocked order states

### Risk & quant analytics
- Historical VaR (95/99), expected shortfall, annualized vol
- Per-position beta, historical vol, max drawdown, position VaR
- Portfolio beta to SPY and constraint breach count
- Institutional quant readiness score and process-layer breakdown
- Bias controls, data quality, point-in-time readiness subscores
- Pending CIO risk reviews (VaR breach, theme weight breach, etc.)

### Deterministic operators & gates
- Operator readiness (pass/fail/review counts)
- Per-operator status, score, confidence, and blocked actions
- Macro regime, gold thesis, concentration, catalyst, archive mismatch, mandate, cash-fortress evaluations
- Execution safety confirmation

### Thesis & forecasting
- Thesis lifecycle states (active, confirmed, watch, etc.)
- Per-thesis probability, confidence, priority, and evidence summaries
- Portfolio target weights (research-only, no orders)
- Superforecast snapshot ID and 90-day directional targets
- Brier accountability and resolved accuracy by horizon
- Method comparison (house vs analyst consensus)
- Forecast coverage across watchlist names

### Market tape & cross-asset context
- Index moves (SPY, QQQ, IWM, RSP)
- Volatility proxy moves (VXX, UVXY)
- Dollar and rates proxy moves (UUP, TLT, IEF, SHY)
- Gold, silver, miner ETF confirmation
- Sector ETF rotation table
- Credit/liquidity stress proxies (HYG, JNK, LQD, AGG)
- Derived cross-market scores (risk appetite, rotation, breadth, gold confirmation, etc.)
- Active interpretation flags (tech selloff, AI thesis failure, quantum panic, EM risk-off, dollar pressure, bank thesis, etc.)
- VIX and Fear & Greed readings (with staleness handling)
- Macro and institutional sentiment tone
- Top price movers and premarket vs regular-session behavior
- Strongest/weakest theme baskets (ECE-style)

### Capital flow & institutional microstructure
- Per-ticker institutional bias (inflow, outflow, distribute, accumulate, neutral)
- Main-force net flow by lot size (super-large, large, medium, small)
- Capital flow on holdings and watchlist names
- Flow collision summaries at portfolio/market level (ACMS)

### News, sentiment & catalysts
- Fresh headline tape by source and age
- Ticker sentiment scores and labels
- Latest news link evidence report (with accountability status)
- Missing/broken URL counts
- Forward catalyst calendar (earnings, conferences, CEO appearances, etc.)
- Calendar intelligence gaps (investor days, analyst days, named events)
- Unexplained large-move flags

### Analyst & fundamental overlays
- Analyst target ranges (low/avg/high) and buy/hold/sell counts
- Fundamentals freshness indicators
- 8-lens scoring per holding and watchlist rank
- Per-ticker multi-lens breakdown (L1–L8 components)

### CIO cognition & manual authority
- Latest cognition journal ID and timestamp
- Strategic thinking, planning, and execution intent records
- Manual sitrep and prediction archives
- Regime–cognition alignment delta
- Linked theses and key risks from manual entries
- CIO strategy update footer (highest tactical authority block)
- Follow-up resolution prompts

### PEI (prospective event intelligence)
- Active geopolitical/macro event pathways
- Scenario trees with branch probabilities
- Event-to-sleeve transmission maps
- Allowed vs blocked actions per scenario branch
- Event resolution dates

### ACMS-COP (cross-market confirmation)
- Dominant behavioral state counts (accumulation, distribution, unclassified)
- Per-ticker/theme flow collision classification
- Multi-horizon forecast stack (choppy digestion, hawkish fade, relief rally, yen carry, credit accident)
- Second tranche and scale-in block status
- Agent accountability summary (9 agent postures)

### NITE-PEI (Bayesian thesis engine)
- CKRI score and zone (critical/warning/etc.)
- Kill condition breakdown with weights and confirmed/inactive states
- Weighted kill contributions by thesis
- Correlation penalty and manual execution flags
- Kelly sizing advisory (research layer)

### BGTM-V1 (game theory & Nash equilibrium)
- N-player Nash equilibrium analysis with multiplicity resolution
- Quantal Response Equilibrium (QRE) outcome distributions
- Correlated equilibrium envelope over surviving equilibria
- Global games threshold / kill-condition classification
- Geo-LR bridge outputs into NITE-PEI Bayesian updating

### Monitoring, archive & operations
- Alert feed by severity, layer, and type
- Data lineage and dataset SHA references
- Source health (active count, tier breakdown, SLA breaches)
- Per-section freshness ages
- Dataset archive snapshot count and recovery status
- Historical backfill queue status
- CIO decision ledger pending review queue
- Execution intelligence / TCA readiness (read-only doctrine)
- Transaction cost analysis placeholders

### Contradictions & accountability
- Formal contradiction register (forecast, thesis, news, governance conflicts)
- Source accountability warnings per news item
- Integration status across ACMS / NITE-PEI / news layers
- Readiness change categories (regime, governance, operators, PEI, freshness, Brier)

### Research QA & institutional grade
- Consistency audit score
- ECE renderer match checks
- Evidence mapping validation
- Causal status logic check
- Execution safety gate
- Freshness gate warnings
- Final institutional grade score

### Shadow / experimental layers
- Cheap-talk / credibility signals
- VWAP proxy positioning
- OPEX calendar proximity
- Geo-game-theory LR bridge outputs
- SLICDO learning spine open-item counts

### Structured workbook (XLSX) extras
- Law/governance binding tables
- PEI event/scenario/sleeve/playbook/Brier tabs
- STR signal entropy, cost basis, Kelly, hedge review
- V3 canonical reconciliation, contract, pipeline, replay, benchmark tabs
- Open order state, artifact manifest, risk overlay
- Capital flow, monitoring, report QA, NITE-PEI summary tabs
- CIO manual archive listings

### Published companion surfaces (related, not the TXT body)
- GitHub dashboard data bindings
- Chief strategist context payload
- Public vs raw dataset views

## Quick start

```bash
mkdir bluelotus-lab && cd bluelotus-lab
python -m bluelotus_engine.cli init-workspace
python -m bluelotus_engine.cli pipeline --once --dry-run
python -m bluelotus_engine.cli clerk
```

Uses **synthetic demo data** — no API keys required.

---

## CLI

| Command | What it does |
|---------|----------------|
| `init-workspace` | Create `.env`, folders, sample dataset |
| `pipeline --once` | Governance → report → clerk → NITE-PEI → SLICDO |
| `clerk` | Deterministic Zone A contradiction cycle |
| `governance-gate` | Run governance gate on dataset |
| `nite-pei` | Bayesian thesis update |
| `slicdo` | Institutional learning cycle |
| `validate` | Post-generation output checks |

If `bluelotus` is not on PATH: `python -m bluelotus_engine.cli <command>`
